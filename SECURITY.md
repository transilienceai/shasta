# Security Policy

## Reporting a vulnerability

If you discover a vulnerability in this repository, report it privately to **contact@transilience.ai**.

Do not open a public issue for:

- credential leaks
- insecure defaults
- dependency or supply chain issues
- vulnerabilities in build or release automation

## What to include

- affected files or workflow
- impact summary
- reproduction steps
- suggested mitigation if you have one

## Scope

This policy covers the code, automation, documentation, and packaged artifacts in this repository.

It does not cover vulnerabilities found in third-party systems you assess with this tool. Follow the target owner's disclosure policy for those findings.
